# Plan: Null-safe IoT Data (Issue #2)

## Problem
`getSolarData` and `getSatelliteData` in `src/routes/iot.ts:21-43` don't handle null/undefined inputs or NaN intermediates. If `projectId` is invalid or `seededRandom` returns NaN, calculations propagate NaN silently.

## Changes

1. **Import logger** from `../lib/logger`
2. **Add default constants**: `DEFAULT_EFFICIENCY_PCT = 60`, `DEFAULT_FOREST_DENSITY_PCT = 50`
3. **Guard `getSolarData`**:
   - Check if `projectId` is NaN → log warning, return defaults
   - Check if `base`/`drift` are NaN → log warning, use 0
   - Final output uses `??` fallback for NaN results
4. **Guard `getSatelliteData`**:
   - Same pattern as above with forest density defaults
